# IO-Feedback-Sync

본 프로젝트는 외부 라이브러리를 일절 배제하고, ATmega328P MCU의 디지털 I/O 포트 제어 시 발생하는 하드웨어적 전파 지연을 나노초(ns) 단위로 분석하여 소프트웨어적으로 동기화를 달성한 프로젝트입니다.
단순히 핀의 출력을 바꾸고 읽는 것을 넘어, 데이터시트의 디지털 I/O 내부 회로도와 타이밍 다이어그램을 바탕으로 out 명령어 직후 in 명령어를 실행할 때 발생하는 '1클럭의 시차'를 하드웨어 레벨에서 검증하고, _NOP() 단 한 줄로 최적의 동기화 래치 타이밍을 확보하는 아키텍처 중심의 펌웨어를 구현했습니다.

## Project Overview
1.  나노초(ns) 단위의 클럭 주기 및 전파 지연 분석16MHz 외부 크리스탈 환경에서 ATmega328P CPU의 1클럭 사이클은 62.5ns입니다. 소프트웨어가 레지스터의 비트를 조작하는 속도와 전기가 실제 내부 플립플롭, 슈미트 트리거, 동기화 래치를 거쳐 되돌아오는 물리적 속도의 차이를 분석하여 시스템의 타이밍 마진을 완벽하게 통제했습니다.
2. 하드웨어 동기화 회로의 이해물리적 핀의 전압 변화가 CPU의 입력 레지스터(PINx)에 안전하게 안착하기까지 거치는 [슈미트 트리거 -> D-L -> PINxn D-FF] 구조를 파악했습니다. 메타스태빌리티를 방지하기 위해 설계된 하드웨어 제약을 이해하고, 이를 우회하기 위한 소프트웨어 타이밍 밸런스를 맞추었습니다.
3. 의도적인 레벨 래치 제어를 통한 Feedback Loop 구현출력 핀으로 내보낸 데이터가 물리적 노드를 거쳐 최신 상태로 업데이트되는 시점을 정확히 저격하여, 입력 핀(PINB0)으로 상태를 되읽어온 뒤 내부 LED와 실시간 동기화하는 하드웨어 피드백 루프를 완성했습니다.

- 일반적인 디지털 I/O 회로에서 PORTx(출력)에 쓴 값이 PINx(입력) 레지스터에 안착하기까지 다음 경로를 거침니다.
  - 출발: PORTxn D-FF
  - 경로: 출력 버퍼 -> 실제 외부 핀(Pxn) -> 슈미트 트리거 -> D-L
  - 도착: PINxn D-FF

## 타이밍 분석
<img width="677" height="288" alt="image (5)" src="https://github.com/user-attachments/assets/aa34c1ec-aeda-4abb-875c-62a3592cbb36" />

- out 명령어로 데이터를 내보낸 직후 in 명령어로 읽을 때 발생하는 1클록의 시차 분석
- 명령어 실행 흐름
  - out PORTx, r16
    - CPU가 WRx 신호를 활성화 해 데이터를 PORTxn D-FF 입구에 전달합니다.
  - nop
    - 상승 에지에서 PORTxn의 Q값이 업데이트되어 실제 핀으로 전기가 출력됩니다.
    - 상승 레벨에서 전기가 한 바퀴 돌아와 SYNC LATCH 문이 열리며 데이터가 안착합니다.
  - in r17, PINx
    - 다음 clk 주기가 시작될 때 상승 에지 때 PINxn D-FF의 Q값이 최신 데이터로 업데이트 됩니다.
    - 명령어가 실행되어 RPx 신호가 버퍼를 열어 CPU가 최신 데이터를 안정적으로 읽어 갑니다.
    - 이제 다음 clk 주기 때 CPU가 r17에 데이터를 write 해 최종적으로 r17에 값이 업데이트 됩니다. 
  
## _NOP() 명령어 필연적
만약 out 명령어 바로 다음 줄에 in 명령어를 배치하면, CPU가 PINx를 읽으려고 버퍼를 여는 시점에 PINxn D-FF는 아직 이전 주기의 과거 데이터를 쥐고 있습니다.
따라서 중간에 CPU를 딱 1사이클(62.5ns) 동안 쉬게 만드는 _NOP() 명령어를 강제로 삽입해야 합니다. CPU가 쉬는 그 한 박자(High-Low 레벨 구간) 동안 외부 핀을 찍고 돌아온 전하가 SYNC LATCH를 통과해 PINxn D-FF 입력단에 쓸 준비를 마칠 수 있으며, 그 다음 상승 에지에서 최신 피드백 데이터를 안전하게 낚아챌 수 있게 됩니다.

## 프로젝트 후기 및 느낀 점
1. 나노초(ns) 단위 타이밍 제어
단순히 코드상에서 연속된 줄에 위치한 쓰기(PORTx)와 읽기(PINx) 명령어가 하드웨어 내부 공용 시스템 데이터 버스 및 플립플롭 회로망을 거칠 때, 물리적인 시차(62.5ns)가 발생한다는 사실을 시각적으로 완전히 이해했습니다. 감에 의존하는 대형 delay 함수 대신 정확히 1사이클을 비워주는 _NOP()를 통해 완벽한 타이밍 매칭을 성공시켰을 때 성취감을 느꼈습니다.

2. 소프트웨어와 하드웨어 아키텍처의 유기적 결합
대학교 실습 시절 핀 제어 도중 간헐적으로 오작동하거나 핀 상태 읽기가 씹히던 현상들의 근본적인 원인이 하드웨어 메타스태빌리티 방지용 2단 동기화 구조 때문이었다는 것을 데이터시트 정주행을 통해 스스로 증명해 냈습니다. 이론으로만 존재하던 데이터시트의 타이밍 다이어그램을 소스코드로 한 줄씩 번역해 내는 귀중한 경험이었습니다.

3. 최적화에 대한 관점 변화
효율적인 펌웨어 설계란 무조건 대기 시간을 없애는 것이 아니라, 하드웨어 회로가 물리적인 연산과 충전을 완료할 수 있도록 "의도된 물리적 여백을 정확한 자리에 제공하는 것"임을 배웠습니다. 시스템 자원을 낭비하지 않는 최소한의 나노초 단위 제어 기술이야말로 로우 레벨 개발자가 가질 수 있는 최고의 무기임을 확신하게 되었습니다.
